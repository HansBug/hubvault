# plan 目录说明

`plan/` 用于存放项目计划、设计与演进文档，但不同阶段的文档需要分目录管理，避免把“从无到有的基础设计”和“后续迭代计划”混在一起。

当前目录约定如下：

- `plan/init/`：项目初始化阶段的基础方案文档，描述从零开始的总体设计、协议、磁盘格式、API 边界和可执行 phase 路线图

后续建议新增的子目录示例：

- `plan/rfc/`：针对某个重大设计变更的专题提案
- `plan/phase-1/`：进入实现后某一阶段的任务分解和交付计划
- `plan/bench/`：性能压测与对比方案
- `plan/ops/`：运维、诊断、迁移、恢复相关计划

原则：

- `plan/init/` 文档尽量保持“初始设计基线”的稳定性，但必须显式对齐当前仓库真实状态，不能假设尚未存在的实现已经落地
- 执行型计划文档应明确 MVP 切分、phase 顺序，以及每个 phase 下的 Todo / Checklist
- `plan/` 下用于执行的任务清单统一使用 Markdown 复选框格式 `* [ ]`
- 后续演进尽量新增新目录或新文档，不直接覆盖这批初始化文档的历史语义
